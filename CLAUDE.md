## Codebase Memory

A persistent memory file exists at `.claude/codebase-memory.md`. It contains a structured
summary of this codebase — architecture, entry points, key patterns, gotchas, and module
breakdowns — generated by a prior deep exploration session.

**At the start of every session:**
1. Read `.claude/codebase-memory.md` before exploring any source files
2. Use it to orient yourself — skip re-reading files already summarized there
3. If the file doesn't exist, or you encounter something that contradicts it, follow the
   Exploration Protocol below

---

## Exploration Protocol

Run this when `.claude/codebase-memory.md` is missing, outdated, or incomplete:

1. Read all root-level orientation files (README, manifests, Makefile, CI config, Dockerfile)
2. Map the top two levels of the directory tree
3. Identify and read entry points for each service or major module
4. Extract key patterns: config, error handling, auth, data layer, observability, testing
5. Note gotchas, non-obvious conventions, and critical env vars
6. Write a fresh `.claude/codebase-memory.md` using the schema defined in
   `.claude/commands/index-codebase.md`

Or simply run the slash command: `/project:index-codebase`

---

## Memory Freshness

The memory file may be stale if:
- A new service or major module was added
- A significant refactor changed file structure or conventions
- Core dependencies were upgraded
- The data layer or auth mechanism changed

When in doubt, check the `_Indexed:` date at the top of `.claude/codebase-memory.md` and
use judgment based on how active this repo is.
